# ForgeCode — 个人简历项目经历

## ForgeCode | AI Coding Agent CLI

**技术栈：** Python 3.10+ / OpenAI SDK / Anthropic SDK / prompt_toolkit / TOML / pytest

**项目背景：** 独立开发的本地命令行 AI 编码助手，对标 Claude Code / Cursor Agent。通过连接大语言模型（OpenAI 兼容接口 / Anthropic API），为开发者提供代码阅读、搜索、编辑和命令执行能力，采用 5 包分层架构（cli / runtime / tools / safety / memory），~6,700 行产品代码 + ~3,200 行测试代码。

- **设计并实现 3 层渐进式上下文压缩引擎**：针对长对话上下文溢出问题，设计截断（同步裁剪 + 磁盘溢出）→ 剪枝（反向扫描零 LLM 开销替换旧工具输出）→ 语义压缩（LLM 生成 7 节结构化摘要，延迟执行不阻塞交互）三层流水线，支持 258K token 上下文窗口的高效利用与递归精简。

- **构建 5 层权限规则链安全体系**：实现路径沙箱 → 计划模式只读 → 破坏性操作策略（DENY/ASK/ALLOW）→ Shell 命令风险分级（安全白名单 + 注入检测 + 敏感环境变量清除）→ 默认允许的链式权限校验，对 8 个内置工具按 READONLY / DESTRUCTIVE / CONCURRENT_SAFE 三级安全标签分类管控。

- **实现只读工具并行执行引擎**：将连续的 READONLY 工具调用自动分组为并行批次，通过 ThreadPoolExecutor 三阶段流水线（串行权限检查 → 并行执行 → 串行按序收集）保证正确性；DESTRUCTIVE 工具自动刷新批次单独串行，兼顾安全与性能。

- **开发计划模式与会话/记忆持久化系统**：实现结构化只读规划阶段（4 种执行策略供用户选择），会话双轨持久化（完整日志 + 压缩上下文快照，原子保存），以及基于文件的项目隔离记忆系统（4 种记忆类型 + YAML 前置元数据 + 自动索引），支持跨会话上下文延续。

- **搭建完整工程化体系**：内置 SWE-bench 评测流水线（选取 → 推理 → 评分 → 报告一键执行），10 个测试文件覆盖全部核心模块（配置合并、权限沙箱、3 层压缩、并行执行线程安全与排序保证、ESC 取消与流式中断恢复等），支持 OpenAI 兼容和 Anthropic 双后端含 Extended Thinking。

## ForgeCode | AI Coding Agent CLI（STAR 版）

**技术栈：** Python 3.10+ / OpenAI SDK / Anthropic SDK / prompt_toolkit / TOML / pytest

**项目背景：** 现有 AI 编码助手（如 Claude Code、Cursor Agent）均为商业闭源产品，开发者难以定制工具链或控制数据流向。为此独立开发了 ForgeCode——一个本地运行的命令行 AI 编码助手，采用 5 包分层架构（cli / runtime / tools / safety / memory），~6,700 行产品代码 + ~3,200 行测试代码。

- **[S]** Agent 在长对话中频繁调用工具，单次工具输出可达数万行，导致 258K token 上下文窗口快速耗尽，对话被迫中断。**[T]** 需要在不丢失关键信息的前提下，最大化上下文窗口利用率，且压缩过程不能阻塞用户交互。**[A]** 设计并实现 3 层渐进式压缩引擎：Layer 1 截断同步裁剪超大输出并溢出到磁盘可回溯；Layer 2 剪枝以零 LLM 开销反向扫描替换旧工具输出为占位符；Layer 3 语义压缩通过 LLM 生成 7 节结构化摘要，采用延迟执行策略在下一轮开始时处理，不阻塞热路径，并支持递归精简过长摘要。**[R]** 实现了 258K token 上下文窗口的高效利用，长对话场景下 Agent 可持续工作而不中断，前两层零额外 LLM 开销，第三层仅需一次异步 LLM 调用。

- **[S]** AI Agent 直接操作用户文件系统和 Shell 存在安全风险：误删文件、执行危险命令、访问沙箱外路径、泄露 API Key 等敏感环境变量。**[T]** 需要构建细粒度的安全管控体系，在保证 Agent 能力的同时防止越权操作。**[A]** 构建 5 层权限规则链：路径沙箱限制文件访问范围 → 计划模式强制只读 → 破坏性操作三级策略（DENY 拦截 / ASK 确认 / ALLOW 放行）→ Shell 命令风险分级（安全命令白名单 + 子命令识别 + 注入检测）→ 默认允许兜底；同时对 8 个内置工具按 READONLY / DESTRUCTIVE / CONCURRENT_SAFE 三级安全标签分类，并在命令执行前自动清除含 API_KEY、SECRET、TOKEN 等模式的环境变量。**[R]** 实现了从文件路径到 Shell 命令的全链路安全管控，在默认 ASK 模式下每个破坏性操作均需用户确认，杜绝了越权访问和敏感信息泄露风险。

- **[S]** Agent 单轮对话中常需连续调用多个只读工具（如同时搜索多个文件、读取多个目录），串行执行导致响应延迟线性增长。**[T]** 在保证结果正确性和安全性的前提下，减少多工具调用的总耗时。**[A]** 实现只读工具并行执行引擎：自动将连续的 READONLY 工具调用分组为并行批次，通过 ThreadPoolExecutor 三阶段流水线执行——串行权限检查确保安全 → 最多 4 线程并行执行提升吞吐 → 串行按原始顺序收集结果保证一致性；遇到 DESTRUCTIVE 工具自动刷新当前批次转为串行执行。**[R]** 多工具并行场景下响应耗时显著降低，同时通过 12 种分组场景测试和线程安全测试保证了并行执行的正确性与结果排序一致性。

- **[S]** 复杂编码任务直接执行容易方向偏离或产生不可逆修改；同时 Agent 每次启动都丢失上次对话的上下文和用户偏好，需要重复沟通。**[T]** 需要支持"先规划后执行"的工作流，并实现跨会话的状态和记忆延续。**[A]** 开发计划模式：通过 `/plan` 命令进入只读规划阶段，安全规则自动切换为仅允许写入计划文件，退出时提供 4 种执行策略供用户选择；实现会话双轨持久化——完整消息日志与压缩上下文快照独立存储、原子保存；构建基于文件的项目隔离记忆系统，支持 4 种记忆类型（用户画像 / 行为反馈 / 项目信息 / 外部引用），YAML 前置元数据 + MEMORY.md 自动索引，记忆内容注入系统提示词。**[R]** 复杂任务通过先规划再执行降低了误操作风险，会话恢复和跨会话记忆使 Agent 能延续之前的工作上下文和用户偏好，减少重复沟通。

- **[S]** 项目需要支持多种 LLM 后端（DeepSeek、OpenAI、Claude 等），且缺乏系统化的质量评估和回归验证手段。**[T]** 建立统一的模型抽象层和完整的工程化体系，确保多后端兼容与代码质量可量化。**[A]** 设计 ModelClient 协议统一抽象 OpenAI 兼容和 Anthropic 两种后端（含 Extended Thinking 和 Thinking Blocks 签名保留）；内置 SWE-bench 评测流水线（选取 → 推理 → 评分 → 报告一键执行），支持断点续跑和分层抽样；编写 10 个测试文件覆盖全部核心模块，包括配置合并、权限沙箱、3 层压缩、并行执行的 12 种分组场景和线程安全验证、ESC 取消与流式中断恢复等。**[R]** 实现了通过配置一键切换模型后端，SWE-bench 评测流水线可量化 Agent 编码能力，测试套件覆盖全部核心路径，保障了迭代过程中的代码质量与回归安全。
